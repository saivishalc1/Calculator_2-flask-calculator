What is SOC

Separation of concerns is a really important fundamental part of software development. The principle is that it is important to not write your code in huge blocks of code, and to instead break up the code into little chunks so that each tiny part of the code will be able to complete each distinct job. It is a design pattern that separates each section to address a separate concern. The benefit of SOC is that it makes the code more stable and allows for increased maintainability, and there are numerous other benefits.
